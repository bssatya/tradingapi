Conventions¶

C++¶

Compiler Version¶

Our code must be C++11 compliant.

We aim to be conformant on a number of platforms

  • Linux / GCC

    • Active development.

  • macOS / Clang

    • Active development.

  • Windows / Visual C++

    • No active development. It may work, but no guarantees given.

Code Style¶

As a general principle, all our code has to conform to the Google Style Guide.

To ensure compliance, as part of our development process, we check our code using cpplint. We do allow a few exceptions, though

  • filter=-runtime/references

    • We prefer passing references to pointers, e.g. foo(const Bar &) is preferred to foo(const Bar *).
    • The reason is that we don’t want run–time checks for nullptr.

  • filter=-build/c++11

    • Google disallows the use of e.g. #include <thread> and #include <chrono>.
    • We prefer to adopt these new features.

Testing¶

We use googletest as the test runner for all our test cases.

Command–Line Flags¶

We use gflags to parse command–line arguments.

Logging¶

We use glog for all logging. Except for some thirparty dependencies, we will never write directly to stdout or stderr. The idea is that you should be able to configure your desired logging style from your environment.

Our own preferences are

export GLOG_logtostderr=1
export GLOG_minloglevel=0

Documentation¶

We use Sphinx and Read the Docs to generate our documentation.

C++¶

We use Doxygen and Breathe to automatically parse and prepare C++ code documentation.
